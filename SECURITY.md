# Security Policy

## Supported versions

| Version | Supported |
|---|---|
| 1.x | ✅ |
| < 1.0 | ❌ |

## Reporting a vulnerability

**Please do not report security vulnerabilities in the public issue tracker.**

If you discover a vulnerability in TactileCs, please send an e-mail to
**security@graph-technologies.io** (or, if that address is unavailable, open a
[private security advisory](https://github.com/graph-technologies/tactile-cs/security/advisories/new)
directly on GitHub).

Please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce (a minimal proof-of-concept is helpful but not required).
- Any suggested mitigation or fix if you have one.

We will acknowledge your report within **72 hours** and aim to release a patch
within **14 days** for critical issues.

## Disclosure policy

We follow a coordinated disclosure process:

1. You report the vulnerability privately.
2. We investigate and develop a fix.
3. We release a patched version and publish a [security advisory](https://github.com/graph-technologies/tactile-cs/security/advisories).
4. You may disclose the details publicly after the advisory is published (typically 7 days after the patched release).

Thank you for helping keep TactileCs and its users safe.
