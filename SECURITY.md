# Security Policy

## Supported versions

| Version | Supported |
|---|---|
| 1.x | ✅ |
| < 1.0 | ❌ |

## Reporting a vulnerability

**Please do not report security vulnerabilities in the public issue tracker.**

We strongly prefer that vulnerabilities are reported using
[GitHub's private Security Advisory feature](https://github.com/graph-technologies/tactile-cs/security/advisories/new).
This keeps the disclosure confidential until a fix is available and allows us to
collaborate directly with you through GitHub's built-in tooling.

If you are unable to use GitHub's Security Advisory feature, you may contact us
by e-mail at **tech@graphtechnologies.xyz**. For other ways to reach us, visit
our website at **[graphtechnologies.xyz](https://graphtechnologies.xyz)**.

Please include the following in your report:

- A description of the vulnerability and its potential impact.
- Steps to reproduce (a minimal proof-of-concept is helpful but not required).
- Any suggested mitigation or fix if you have one.

We will acknowledge your report within **72 hours** and aim to release a patch
within **14 days** for critical issues.

## Responsible disclosure policy

We follow a coordinated (responsible) disclosure process:

1. You report the vulnerability privately via GitHub Security Advisories or the contact details above.
2. We investigate and develop a fix.
3. We release a patched version and publish a [security advisory](https://github.com/graph-technologies/tactile-cs/security/advisories).
4. You may disclose the details publicly after the advisory is published (typically 7 days after the patched release).

We kindly ask that you do not publicly disclose the vulnerability before a patch
has been released.
